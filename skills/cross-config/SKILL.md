# cross — AI Agent Safety Configuration

You are helping the user install and configure [cross](https://github.com/rfdearborn/cross), a lightweight safety harness for AI agents. cross monitors tool calls through a 3-layer pipeline: deterministic denylist rules, LLM-powered review of flagged calls, and periodic sentinel monitoring.

## What to Do

Walk the user through these steps. Skip steps that are already complete. Be concise.

### Step 1: Install cross

Check if `cross` is on PATH. If not, install it:

```bash
pip install cross-ai
# or with Slack and email support:
pip install cross-ai[slack]
```

### Step 2: Run interactive setup

```bash
cross setup
```

This wizard will:
- Detect OpenClaw on PATH
- Configure LLM providers for the gate reviewer (default: `anthropic/claude-code/claude-sonnet-4-6`) and sentinel (default: `anthropic/claude-code/claude-opus-4-6`). The `anthropic/claude-code/*` provider routes through Claude Code — no API cost for Pro/Max users
- Optionally configure desktop notifications (macOS), email notifications, and Slack
- Install a shell wrapper so `openclaw` automatically runs through cross
- On macOS: optionally patch the OpenClaw Gateway LaunchAgent and auto-start the daemon
- Configure automatic updates

### Step 3: Verify the integration

Start the daemon and test:

```bash
cross daemon &          # start in background (or it may already be running via launchd)
cross wrap -- openclaw  # run OpenClaw through cross
```

Visit `http://localhost:2767` to see the dashboard.

### Step 4: Customize rules (optional)

Users can add custom denylist rules in `~/.cross/rules.d/`. Example:

```yaml
# ~/.cross/rules.d/my-rules.yaml
rules:
  - name: no-docker-push
    tools: [Bash]
    field: command
    action: block
    description: Prevent pushing Docker images
    patterns:
      - 'docker\s+push\b'

# Disable a default rule by name
disable:
  - destructive-rm
```

Rule fields:
- `name` — unique identifier
- `tools` — tool names to match (`[Bash]`, `[Write, Edit]`, `["*"]` for all)
- `field` — input field to check (`command`, `file_path`, or omit for all)
- `action` — `review` (LLM decides), `escalate`, `block`, `alert`, or `halt_session`
- `patterns` — regex patterns (case-insensitive)
- `contains` — substring matching (case-insensitive)

### Step 5: Custom instructions (optional)

Create `~/.cross/instructions.md` with project-specific context for the gate and sentinel LLMs. This file is included in every review prompt, so the LLM understands your project's norms.

## Key Configuration

All settings use the `CROSS_` prefix, stored in `~/.cross/.env` (generated by setup) or `~/.cross/local.env` (personal overrides that survive re-setup):

| Setting | Default | Description |
|---------|---------|-------------|
| `CROSS_LISTEN_PORT` | 2767 | Proxy/dashboard port |
| `CROSS_LLM_GATE_MODEL` | anthropic/claude-code/claude-sonnet-4-6 | LLM for reviewing flagged calls |
| `CROSS_LLM_GATE_ENABLED` | true | Enable LLM review |
| `CROSS_LLM_GATE_SHADOW` | false | Shadow mode: LLM decides but human makes final call |
| `CROSS_LLM_SENTINEL_MODEL` | anthropic/claude-code/claude-opus-4-6 | LLM for periodic session monitoring |
| `CROSS_LLM_SENTINEL_ENABLED` | true | Periodic session monitoring |
| `CROSS_LLM_SENTINEL_INTERVAL_SECONDS` | 60 | Seconds between sentinel reviews |
| `CROSS_GATE_APPROVAL_TIMEOUT` | 300 | Seconds to wait for human approval |
| `CROSS_AUTO_UPDATE_ENABLED` | true | Auto-update cross |

Supported LLM providers: `anthropic/claude-code/<model>` (default, free for Pro/Max), `google/gemini-3-flash-preview`, `anthropic/claude-haiku-4-5`, `openai/gpt-4o`, `ollama/llama3`.

## Troubleshooting

- **Daemon not running**: `cross daemon` to start, or check `launchctl list | grep cross` on macOS
- **Hook not loading**: Verify `NODE_OPTIONS` includes `--import .../openclaw_hook.mjs` — `cross wrap -- openclaw` sets this automatically
- **Dashboard blank**: Confirm daemon is running on the expected port (`curl http://localhost:2767/cross/api/status`)
- **LLM gate slow**: Try a faster model or disable with `CROSS_LLM_GATE_ENABLED=false` (denylist rules still active)
